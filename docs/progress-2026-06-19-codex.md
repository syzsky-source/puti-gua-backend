# 菩提卦馆 2026-06-19 Codex 工作进度

> 下次用户说“继续今天 Codex 进度”或“继续 6 月 19 日 Codex 进度”时，从本文件继续。

## 已完成并已上线

### 1. Codex 功能改造已合并

- 前台用户 ID 使用浏览器 `localStorage` 中固定的 `puti_user_id`。
- 同一浏览器后续购买、问卦会使用同一个用户 ID。
- 购买弹窗支持显示当前客户 ID、复制客户 ID、主动生成新客户 ID。
- 创建订单时会绑定当前用户 ID 与所选套餐。
- 后台订单支持按状态、套餐、用户 ID、关键词筛选。
- 后台确认付款后，会给订单绑定的用户自动增加对应次数。
- 收款码支持绑定套餐 `product_id` 与金额 `amount`。

### 2. 前台结构与品牌

- 前台已参考 `https://putiyuan.pages.dev/` 调整为九项服务入口。
- 项目对外名称已统一为「菩提卦馆」。

### 3. 数据库与二维码套餐匹配

线上数据库已完成：

- `qrcodes.product_id` 字段。
- `qrcodes.amount` 字段。
- 套餐二维码查询索引。
- 套餐与三张二维码绑定。

正式套餐以以下三档为唯一标准：

```text
10次问卦 / ¥19.90 / qrcode_19_90.jpg
30次问卦 / ¥29.90 / qrcode_29_90.jpg
100次问卦 / ¥69.90 / qrcode_69_90.jpg
```

正式产品 ID：

```text
p_9_10
p_19_30
p_49_100
```

二维码地址：

```text
https://api.putiguaguan.fun/uploads/qrcodes/qrcode_19_90.jpg
https://api.putiguaguan.fun/uploads/qrcodes/qrcode_29_90.jpg
https://api.putiguaguan.fun/uploads/qrcodes/qrcode_69_90.jpg
```

### 4. 已完成线上验收

用户已测试通过完整流程：

```text
前台选择套餐
→ 显示对应收款码
→ 创建订单
→ 上传付款截图
→ 后台看到订单
→ 后台确认付款
→ 正确用户自动增加对应次数
```

后端 PM2 服务当前使用：

```text
puti-gua-backend
```
