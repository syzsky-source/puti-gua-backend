## 一、已完成并已上线

### 1. 充值、订单、加次数主流程

- 前台用户 ID 使用浏览器 `localStorage` 中固定的 `puti_user_id`。
- 同一浏览器后续购买、问卦会使用同一个用户 ID。

### 1. 新用户默认免费次数

目标：**新用户默认只赠送 1 次**。

代码已改为默认 1 次，并新增“项目 `.env` 优先于 PM2 旧环境变量”的处理，避免旧的 `FREE_POINTS=3` 覆盖新配置。

但用户反馈仍有人拿到 3 次，因此下次必须先确认服务器已执行并验证：

```bash
cd /var/www/puti-gua-backend
git pull origin main

if grep -q '^FREE_POINTS=' .env; then
  sed -i 's/^FREE_POINTS=.*/FREE_POINTS=1/' .env
else
  echo 'FREE_POINTS=1' >> .env
fi

node -e "console.log('当前新用户默认次数：', require('./src/config/env').freePoints)"
pm2 restart puti-gua-backend --update-env
pm2 save
```

预期输出：

```text
当前新用户默认次数： 1
```

注意：已创建的测试用户继续保留其原有次数；需要无痕窗口、清站点缓存，或“生成新客户ID”后才可验证新默认值。
